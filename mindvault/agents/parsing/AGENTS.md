# 解析智能体 AGENTS

## 角色

解析智能体负责把原始输入变成结构化候选知识。
用户只应看到“解析智能体”这个一级对象。

## 内部实现约定

- 允许内部包含关系识别、去重、占位补全等步骤。
- 这些步骤属于内部实现，不应暴露成独立主智能体。

## 输出要求

- 输出细粒度、可继续组织的结构
- 把噪声压低
- 优先保留真实信息，不追求表面丰富
- 优先保留“可画像、可比较、可连线”的特征信息，不要只抽对象名
- 需要时可以把一段资料拆成多条：
  - claim
  - relation
  - event
  - entity attribute
- 如果信息明确，宁可多拆字段，也不要少拆。
- 同一段话如果同时包含“对象 + 属性 + 条件 + 结果”，应尽量拆成多条结构，而不是合并成一条宽泛说法。
- 对聊天记录，默认要补出：
  - 说话人
  - 说话对象
  - 主题
  - 立场/偏好
  - 互动动作
  - 资源链接或外部引用
  - 人物画像特征
    - 身份/角色
    - 语气/风格
    - 偏好/倾向
    - 立场/观点
    - 习惯/能力
    - 关系边界
    - 关注点/担忧
    - 常提及对象
- 对技术讨论，默认要补出：
  - 产品/组件
  - 部署方式
  - 运行状态
  - 系统要求
  - 操作建议
  - 风险和限制
- 对所有对象，默认要补出：
  - 别名/代称
  - 状态
  - 特征描述
  - 与谁/什么有关
  - 是否有明确来源片段
- 每次输出前自查：
  - 是否误把数字当实体
  - 是否遗漏 URL / 命令 / 版本 / 配置项
  - 是否把多个小事实挤成了一条大 claim
  - 是否把人物、组织、产品、地点只抽成了“名字”，却漏掉了关键画像和特征
